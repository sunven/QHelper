## Affected Pages/Routes

- `chrome-extension://<id>/tools/text-preview.html` - the new Text Preview workbench route.
- `chrome-extension://<id>/tools.html` with `/tools/text-preview.html` alias - route alias generated by WXT build hooks.
- Popup and side navigation - catalog discovery and tool launch must include `text-preview`.

## Key Interactions to Verify

- Fill sample text and verify `192.168.80.45` and `ls -l $(which kubectl)` are copyable results.
- Paste/edit text in CodeMirror, switch workspace tabs, rename/delete tabs, reload, and confirm workspace restores from `window.localStorage`.
- Navigate away to another QHelper tool and back; Text Preview must unmount while inactive, then restore from persisted workspace.
- Open one or more local files through a user click, save a dirty file, reload disk content, force overwrite after conflict, and create a download fallback copy.
- Copy one result and copy all results; when `navigator.clipboard.writeText` fails, show the manual copy textarea.
- Clear all and verify workspace text plus IndexedDB file handles are removed.

## Edge Cases

- Input over 1 MB hard-stops parsing; input over 200 KB shows a warning and still parses.
- Result limits cap 200 items per type and 1000 total items.
- Parser failure from one parser does not prevent other parser results.
- Password/token-shaped strings are not extracted.
- URL path segments are not duplicated as Unix path results, while command-contained paths remain copyable.
- File System Access unsupported browsers show an unsupported path but still allow paste, parse, and copy.
- `showOpenFilePicker` remains called only from the `打开文件` user action.
- Local file save detects changed disk content before overwrite.
- Storage corruption falls back to a clean workspace and reports a local-save warning.
- Pasted or file-loaded sentinel text appears only in `qhelper.text-preview.workspace.v1`, never in request URL, headers, body, `sessionStorage`, `chrome.storage.local`, or unrelated `localStorage` keys.

## Critical Paths

- Text Preview route registration: metadata -> `ORDINARY_TOOL_IDS` -> generated alias -> `toolRoutes` component.
- Text parsing pipeline: input -> size guard -> parser fan-out -> conflict resolver -> result limits -> grouped result UI.
- Workspace persistence: UI state -> pure workspace reducer -> `window.localStorage` key `qhelper.text-preview.workspace.v1` -> restore after refresh.
- File handles: file open -> IndexedDB DB `qhelper-text-preview-file-handles` -> save/reload/conflict handling -> clear-all cleanup.
- Privacy sentinel: paste/open content -> local parser/storage only -> Playwright request observer sees no sentinel in network traffic.
